🔥 TP9 — تقنيات Tooltip و Modal بدون JavaScript

/* Tooltip — نظرة على :hover */
.tooltip { position: relative; }
.tooltip .tooltip-text {
  visibility: hidden;       /* مخفي افتراضيًا */
  position: absolute;
  bottom: 125%;
}
.tooltip:hover .tooltip-text {
  visibility: visible;      /* يظهر عند التحويم */
}

/* Modal — نظرة على :target */
.modal    { display: none; position: fixed; }
.modal:target { display: block; }  /* يفتح عند النقر على الرابط */
✅
الفكرة الأساسية لـ TP9: استعمال CSS pseudo-classes فقط (بدون JS) لتحقيق تفاعلات مثل Tooltip و Modal. هذا نمط متكرر ومهم.
🚨
الفرق المهم — display:none vs visibility:hidden
display: none → يزيل العنصر تمامًا من التدفق (لا يشغل مساحة)
visibility: hidden → يخفي العنصر لكن يحتفظ بمساحته

💡

🔄 الشروط والتحكم في التنفيذ

if / else / elseif

$note = 12;
if ($note >= 10) {
    echo "Admis";

المرحلة 2 — الأمان: Attaques vs Protection

⭐ مهم جداً
🔥
قال الأستاذ: هناك ~8 هجمات. يجب فهم وظيفة كل هجوم. قد يعطي كود ويسأل ماذا يفعل. لا يطلب كتابة PDO كامل — فقط الفهم.

⚡ 1. SQL Injection — حقن SQL

❌ كود خطير (بدون حماية)

$sql = "SELECT * FROM users 
  WHERE username='$user' 
  AND password='$pass'";
// الهجوم: نكتب في حقل username:
// ' OR '1'='1
// → يتجاوز كلمة المرور!

✅ حماية بـ PDO

$stmt = $pdo->prepare(
  "SELECT * FROM users 
   WHERE username = :u 
   AND password = :p"
);
$stmt->execute([
    ':u' => $user,
    ':p' => $pass
]);
العنصرالوظيفة
prepare()يُفصل الكود SQL عن البيانات — الاستعلام يُحضَّر أولًا بدون بيانات
execute()يُدخل البيانات بأمان — حتى لو تضمنت ' أو " لن تُنفَّذ كـ SQL
:u / :pPlaceholders — أماكن آمنة لاستقبال البيانات

⚡ 2. XSS — Cross Site Scripting

❌ بدون حماية

echo $_POST['comment'];
// المهاجم يُدخل:
// <script>alert('hack')</script>
// → ينفّذ JS في متصفح الضحية!

✅ حماية بـ htmlspecialchars

echo htmlspecialchars(
    $_POST['comment'],
    ENT_QUOTES,
    'UTF-8'
);
// <script> → &lt;script&gt;
// يظهر كنص، لا يُنفَّذ ✅
المعاملالوظيفة
htmlspecialchars()تحوّل < > & ' " إلى entities HTML آمنة
ENT_QUOTESتحمي علامات الاقتباس المفردة والمزدوجة
'UTF-8'ترميز آمن لتجنب مشاكل التشفير

function calculerMoyenne($notes) {
    $total = 0;
    foreach ($notes as $n) {
        $total += $n;   // تراكم المجموع
    }
    return $total / count($notes);  // count() = عدد العناصر
}

$notes = [14, 7, 18, 9, 11];

echo "<ul>";
foreach ($notes as $note) {
    $statut = ($note >= 10) ? "Admis ✅" : "Ajourné ❌";
    echo "<li>$note — $statut</li>";
}
echo "</ul>";

$moy = calculerMoyenne($notes);
echo "<p>Moyenne générale : $moy</p>";
?>

📋 المطلوب

أنشئ قاعدة بيانات BiblioDB تحتوي جدول livres. ثم أنشئ نموذج PHP لإضافة كتاب وعرض قائمة الكتب.

✅ الحل — الجزء SQL
CREATE DATABASE BiblioDB;
USE BiblioDB;

CREATE TABLE livres (
    id      INT         PRIMARY KEY AUTO_INCREMENT,
    titre   VARCHAR(200) NOT NULL,
    auteur  VARCHAR(100) NOT NULL,
    annee   INT         CHECK (annee > 1000)
);

INSERT INTO livres (titre, auteur, annee)
VALUES ('Le Petit Prince', 'Saint-Exupéry', 1943);

SELECT * FROM livres WHERE annee > 2000;
UPDATE livres SET annee = 1944 WHERE id = 1;
DELETE FROM livres WHERE id = 5;
✅ الحل — PHP + PDO (الجزء المهم للامتحان)
try {
    $pdo = new PDO("mysql:host=localhost;dbname=BiblioDB", "root", "");
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

    // إضافة كتاب (prepared statement للحماية)
    $stmt = $pdo->prepare("INSERT INTO livres (titre, auteur) VALUES (:t, :a)");
    $stmt->execute([':t' => "Guerre et Paix", ':a' => "Tolstoï"]);

    // عرض الكتب
    $result = $pdo->query("SELECT * FROM livres");
    foreach ($result as $row) {
        echo "<p>" . htmlspecialchars($row['titre']) . "</p>";
    }
} catch (Exception $e) {
    echo "Erreur: " . $e->getMessage();
}
?>
⚠️ لماذا نستعمل prepare + execute وليس query مباشرة؟
🛡️
query() مع بيانات المستخدم = خطر SQL Injection!
prepare()/execute() يعزل البيانات عن الكود → آمن تمامًا

🔥 أسئلة نظرية مع إجاباتها — الأكثر احتمالًا

س: ما الفرق بين GET و POST في PHP؟
✅ GET: البيانات تظهر في URL، مناسب للبحث وغير آمن للبيانات الحساسة. POST: البيانات مخفية في جسم الطلب، أكثر أمانًا لكلمات المرور والنماذج.
س: ما الفرق بين include و require في PHP؟
✅ include: إذا لم يجد الملف يُعطي Warning ويكمل. require: إذا لم يجد الملف يُعطي Fatal Error ويوقف التنفيذ تمامًا.
س: لماذا نستعمل prepare() بدل استعلام مباشر؟
✅ prepare() يفصل كود SQL عن البيانات، مما يمنع SQL Injection. حتى لو أدخل المستخدم ' OR 1=1 فلن تُنفَّذ كـ SQL.
س: ما وظيفة htmlspecialchars() ؟
✅ تُحوّل الرموز HTML الخاصة (مثل < > & " ') إلى entities HTML آمنة. هذا يمنع هجوم XSS لأن <script> يُعرض كنص ولا يُنفَّذ.
س: ما الفرق بين display:none و visibility:hidden ؟
✅ display:none يُزيل العنصر تمامًا من التدفق ولا يشغل مساحة. visibility:hidden يُخفي العنصر لكنه يظل يشغل مساحته في الصفحة.
س: ما الفرق بين + و ~ في CSS؟
✅ h2 + p يستهدف p الذي يأتي مباشرة بعد h2 (أخ مجاور واحد فقط). h2 ~ p يستهدف كل عناصر p الإخوة التي تأتي بعد h2 (مهما كان عددها).
س: ما هو SQL Injection؟ كيف نحمي منه؟
✅ هجوم يُدخل فيه المهاجم كود SQL في حقل نص (مثل ' OR '1'='1) لتغيير سلوك الاستعلام. الحماية: استخدام PDO مع prepare() و execute() لفصل الكود عن البيانات.
س: ما هو XSS؟ كيف نحمي منه؟
✅ هجوم يُدخل فيه المهاجم <script> في حقل نص لتنفيذ كود JS في متصفح الضحية (سرقة cookies، redirect). الحماية: htmlspecialchars() عند عرض أي بيانات من المستخدم.
س: ما هو Trigger في MySQL؟
✅ Trigger هو كود SQL يُنفَّذ تلقائيًا عند حدوث حدث (BEFORE/AFTER INSERT, UPDATE, DELETE). يستخدم OLD للقيمة القديمة و NEW للقيمة الجديدة. لا يُستدعى مباشرة من PHP.
س: ما الأركان الثلاثة لـ POO؟
✅ Encapsulation (التغليف: حماية البيانات بـ private) + Héritage (الوراثة: class Child extends Parent) + Polymorphisme (تعدد الأشكال: نفس الدالة بتصرفات مختلفة في الأصناف الفرعية).
