المشكلة 2 3 ن
📌 السيناريو:
لديك نموذج بحث في الموقع. المستخدم يُدخل اسمه ويُبحث عنه في قاعدة البيانات. الكود الحالي هو: 
$nom = $_POST['nom'];
$sql = "SELECT * FROM users WHERE nom = '$nom'";
① ما المشكلة الأمنية؟ اشرحها.   ② اكتب الكود الصحيح باستخدام PDO.
⚡ ① المشكلة: SQL Injection
المهاجم يكتب في الحقل: ' OR '1'='1
الاستعلام يصبح: WHERE nom = '' OR '1'='1' → يُرجع كل المستخدمين!
✅ ② الحل — PDO مع prepare/execute: 
// ① الاتصال
$pdo = new PDO("mysql:host=localhost;dbname=mydb", "root", "");

// ② prepare — بدون البيانات الحقيقية
$stmt = $pdo->prepare("SELECT * FROM users WHERE nom = :nom");

// ③ execute — البيانات تُدخَل بأمان
$stmt->execute([':nom' => $_POST['nom']]);

// ④ جلب النتائج
$users = $stmt->fetchAll();
📖 مع PDO، حتى لو كتب المهاجم ' OR '1'='1، سيُعامَل كنص عادي وليس كـ SQL → الحماية الكاملة

السؤال 4 2 ن
ما نوع كل متغير في PHP؟ وما الفرق بين == و === في المقارنة؟ أعطِ مثالاً يوضح الفرق.
المتغيرالنوع
$nom = "Ahmed";String — نص
$age = 25;Integer — رقم صحيح
$prix = 9.99;Float — رقم عشري
$ok = true;Boolean — صح/خطأ
$tab = ["a","b"];Array — مصفوفة
📖 == مقابل ===:
5 == "5" → true (يقارن القيمة فقط — النوع لا يهم)
5 === "5" → false (يقارن القيمة والنوع معاً — رقم ≠ نص)

ننصح باستخدام === دائماً لتجنب أخطاء غير متوقعة.

السؤال 1 1.5 ن
تعريف: ما هو PDO في PHP؟ ما الفرق بين fetch() و fetchAll()؟ ومتى نستخدم كل واحدة؟
✅ PDO (PHP Data Objects): واجهة PHP للتعامل مع قواعد البيانات بشكل آمن. يستخدم prepare() و execute() لفصل كود SQL عن البيانات، مما يمنع هجمات SQL Injection.
الدالةماذا تُرجع؟متى نستخدمها؟
fetch() صف واحد فقط (Array) عندما نتوقع نتيجة واحدة — مثل: البحث بالـ id أو التحقق من مستخدم
fetchAll() كل النتائج (Array من Arrays) عندما نتوقع أكثر من نتيجة — مثل: عرض قائمة الطلاب
السؤال 2 2 ن

① fetchAll() في PDO تُرجع مصفوفة بكل نتائج الاستعلام.

المشكلة 2 3 ن
📌 السيناريو:
مطوّر كتب هذا الكود للبحث عن طالب في قاعدة البيانات: 
$nom = $_GET['nom'];
$sql = "SELECT * FROM etudiants WHERE nom = '$nom'";
$res = $pdo->query($sql);
$etu = $res->fetch();
① ما المشكلة الأمنية في هذا الكود؟ اشرحها بمثال.   ② أعد كتابة الكود بشكل صحيح وآمن باستخدام PDO.
⚡ ① المشكلة: SQL Injection
الكود يضع $nom مباشرة داخل SQL. لو كتب المهاجم في URL: ?nom=' OR '1'='1
الاستعلام يصبح: WHERE nom = '' OR '1'='1' → يُرجع كل الطلاب! خطر جداً.
✅ ② الحل — PDO مع prepare/execute: 
<?php
// ① الاتصال
$pdo = new PDO("mysql:host=localhost;dbname=ecole", "root", "");

// ② prepare — placeholder :nom بدل البيانات الحقيقية
$stmt = $pdo->prepare("SELECT * FROM etudiants WHERE nom = :nom");

// ③ execute — البيانات تُدخَل بأمان
$stmt->execute([':nom' => $_GET['nom']]);

// ④ fetch — نتيجة واحدة (طالب واحد)
$etu = $stmt->fetch();

if ($etu) {
    echo htmlspecialchars($etu['nom']) . " — " . $etu['note'];
} else {
    echo "الطالب غير موجود";
}
?>
📖 مع PDO، حتى لو كتب المهاجم ' OR '1'='1، سيُعامَل كاسم طالب عادي ولن يتأثر الاستعلام.
